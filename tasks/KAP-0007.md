# Build Bundle Verify CLI

Problem: V1's central experience is portable offline verification with precise aggregate verdicts, not a server or workflow-specific command surface.

Acceptance:

- Implement `kapsel bundle verify <package> --trust <policy>` with no network access.
- Make the CLI the composition root that compile-time registers supported `ProfileValidator` implementations.
- Provide concise human output and stable JSON output.
- Print aggregate `VERIFIED`, `FAILED`, `UNTRUSTED`, or `INCOMPLETE` according to KAP-0004; never use ambiguous `PACKAGE: VALID` as the overall result.
- Report structure, cryptography, trust, witnessing, profile, disclosure, and completeness dimensions independently.
- Permit self-consistency/demo mode only when explicit and unmistakably untrusted.
- Use deterministic exit codes aligned with aggregate verdict and parse/config errors.
- Allow profiles to add a bounded named report section without changing generic CLI semantics.

Verification:

- Trusted complete fixture reports `VERIFIED`; tampered, untrusted, unwitnessed, or incomplete fixtures cannot do so.
- Output snapshots and script-consumption tests pass.
- README starts with this command.

## Next action

Ship the verifier CLI before collector or adapter interfaces.
