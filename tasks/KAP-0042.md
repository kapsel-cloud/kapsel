# Build The Public Crash-Recovery Demo

Status: planned; blocked by KAP-0041 source-locality closure.

Problem: deterministic tests and live-kind tests prove complementary facts, but evaluators need one
release-owned demonstration that uses the public application interface and real process termination.

Contract owners: [KAP-0038](../docs/experiments/KAP-0038-kubernetes-effect-gateway-boundary.md),
[testing](../docs/TESTING.md), [build](../docs/BUILD.md), and
[threat model](../docs/THREAT_MODEL.md).

Depends on: [KAP-0041](KAP-0041.md).

## Work

- Build one bounded harness around the supported `kapsel` commands and a disposable owned `kind`
  cluster.
- Demonstrate the healthy image rollout and the unavailable-image `ProgressDeadlineExceeded` result.
- Terminate and restart a real process after the mutation seam and prove no blind second patch.
- Terminate and restart a real process after receipt publication and prove prepared bytes remain
  frozen under changed signing/output settings.
- Inspect the failed receipt offline and show the signed classifier inputs and non-claims.
- Keep fault control outside all agent-facing input.
- Preserve bounded failure logs and remove only resources the harness created.

## Acceptance

- One documented command runs the complete owned demo from setup through cleanup.
- Both owner-required process-termination seams are exercised through the same application interface
  used without faults.
- Provider mutation count, receiver result, receipt digest, and offline inspection are asserted.
- Failure and signal paths still export bounded logs and attempt ownership-safe cleanup.
- The harness refuses ambiguous pre-existing resources, unsupported tool versions, and unavailable
  Docker access before mutation.
- Focused harness tests, `./scripts/ci-local.sh`, the explicit live-kind gate, local Markdown links,
  and `git diff --check` pass.

## Non-goals

No chaos platform, arbitrary fault injection interface, shared cluster support, hosted demo, or
production Kubernetes use.

## Next action

Drive the harness first from the existing mutation and receipt-publication markers, keeping those
markers inaccessible to ordinary command and MCP requests.
