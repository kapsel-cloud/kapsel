# Deepen And Rename The Gateway Module

Status: completed on 2026-07-16.

Problem: the current package and entry-type names repeat the entire experiment description, while
`lib.rs` mixes the public interface, gateway implementation, and most private-seam tests. The
implementation is deep, but the source entry is harder to navigate than necessary.

Contract owners: [architecture](../docs/ARCHITECTURE.md), [technical scope](../docs/SCOPE.md), and
[ADR 0008](../docs/decisions/0008-use-one-kubernetes-effect-gateway-canary.md).

Depends on: the implemented KAP-0038 library foundation.

## Work

- Rename `kapsel-kubernetes-effect-gateway` to `kapsel-gateway`.
- Rename the public entry type from `KubernetesEffectGateway` to `Gateway`.
- Keep concrete scope visible in names such as `SetDeploymentImageRequest`; do not introduce a
  generic request, capability, provider, or plugin interface.
- Keep the repository-only `kapsel-dev` package as the home for development automation, including
  the existing `kapsel-hooks` binary and future tidy checks.
- Make `lib.rs` a compact interface map. Move gateway implementation and its private-seam tests into
  clearly named internal files without changing semantics or vectors.
- Update gateway Cargo metadata, scripts, build documentation, and all focused package selectors
  atomically.

## Acceptance

- Callers use `kapsel_gateway::Gateway` and concrete Kubernetes operation types.
- `lib.rs` exposes the module interface without containing the lifecycle implementation or its large
  test module.
- Authorization, lifecycle, receipt, publication, Kubernetes, and recovery behavior is byte-for-byte
  and state-for-state unchanged.
- No new public seam or crate-per-concept decomposition appears.
- Searches find no stale current references to the old package, directory, or entry-type names.
- `cargo make fmt`, focused tests and Clippy, `./scripts/ci-local.sh`, the explicit live-kind gate,
  local Markdown links, and `git diff --check` pass.

## Non-goals

No evaluator command, application configuration grammar, MCP adapter, new capability, generic core,
or release packaging.
