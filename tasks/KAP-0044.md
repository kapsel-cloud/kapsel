# Package And Document The 0.1.0 Release Artifact

Status: complete.

Problem: implemented commands and adapters are not a release until a fresh evaluator can install an
identified artifact, run the supported journey, understand failure states, and remove owned
resources.

Contract owners: [technical scope](../docs/SCOPE.md), [build](../docs/BUILD.md),
[testing](../docs/TESTING.md), [security](../SECURITY.md), and [privacy](../docs/PRIVACY.md).

Depends on: [KAP-0043](KAP-0043.md).

## Work

- Name the supported Unix target set for `0.1.0`.
- Produce a versioned artifact per supported target with SHA-256 checksums, license, and exact
  source revision.
- Add clean-environment install, version/provenance, operation, restart, inspection, MCP, and
  cleanup smoke tests.
- Publish evaluator setup, operator configuration, expected output, failure-state explanation,
  limits, and architecture diagrams.
- Document prototype-scoped signed formats, unsupported platforms, security boundaries, and every
  material non-claim.
- Define reproducible release assembly without creating a stable evidence-package or plugin
  contract.

## Acceptance

- A clean supported environment installs and runs only from the release artifacts and published
  documentation.
- Artifact digest, license, version, and source revision agree.
- Setup does not require repository-internal commands beyond the separately documented source-build
  path.
- Documentation distinguishes request acceptance, receiver result, `UNKNOWN`, inspection, and
  production-readiness limits.
- Artifact smoke tests cover the local command, offline inspector, MCP adapter, and owned demo.
- `cargo make fmt`, focused tests and Clippy, `./scripts/ci-local.sh`, the explicit live-kind gate,
  artifact smoke tests, local Markdown links, and `git diff --check` pass.

## Non-goals

No stable registry/library contract, automatic updater, Windows support without evidence, hosted
distribution service, or marketing launch. An earlier package distribution does not replace the
owned artifacts, checksums, source-revision proof, or smoke tests.

## Progress

The release contract names only `x86_64-unknown-linux-gnu`, built and smoked in pinned x86-64 Debian
12 containers. Version `0.1.0-rc.1` assembly now produces one normalized archive and adjacent
SHA-256 file with exact source, target, builder, license, and binary provenance. The archive keeps
the ordinary binary separate from the feature-gated demo executable and includes one standalone
evaluator guide.

Artifact smoke verifies dirty-tree refusal, checksum, bounded exact layout, normalized modes,
provenance agreement, safe extraction, grant provisioning, operation and restart, offline
inspection, MCP discovery and call, both demonstration crash seams, frozen receipt bytes under
rotated settings, bounded output, and cleanup using only extracted files. Hosted run
[`29590100498`](https://github.com/kapsel-cloud/kapsel/actions/runs/29590100498) passed the default,
artifact-smoke, byte-for-byte reproducibility, strict clean assembly, and artifact-upload lanes at
revision `623b1bb712a18abadd3749d53ce8b961aacf4c52`. The downloaded archive checksum verified, and
its metadata names version `0.1.0-rc.1`, that exact revision, `source_dirty: false`, and
`x86_64-unknown-linux-gnu`.

On a clean supported x86-64 GNU/Linux checkout of that revision,
`./scripts/test-kind-effect-gateway.sh` passed both healthy and failed-rollout proofs;
`./scripts/demo-kind-crash-recovery.sh` passed the source-built real-process demonstration; and
`python3 scripts/smoke-release-artifact.py --archive <archive> --expected-revision <revision> --live-demo`
passed the same demonstration using only the downloaded artifact. Both demonstrations reported one
provider apply, `ProgressDeadlineExceeded`, a frozen failed receipt under rotated settings, and
offline `INSPECTED`; every owned cluster was deleted. KAP-0044 acceptance is complete.
