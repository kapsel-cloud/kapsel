# Rehearse And Publish 0.1.0

Status: active technical cursor.

Problem: The `0.1.0` release requires one clean-checkout rehearsal and an auditable release
decision, not merely a collection of individually passing development commands.

Contract owners: [KAP-0038 release acceptance](KAP-0038.md), [technical scope](../docs/SCOPE.md),
[build](../docs/BUILD.md), [testing](../docs/TESTING.md), and [review](../docs/REVIEW.md).

Depends on: [KAP-0044](KAP-0044.md).

## Work

- Record version `0.1.0` and tag `v0.1.0` consistently across workspace, artifacts, and notes.
- Rehearse installation, operator setup, healthy operation, both process-termination recoveries,
  failed rollout, offline inspection, MCP invocation, and cleanup from a clean checkout.
- Run every deterministic, live, integration, documentation, and artifact gate on the exact release
  revision.
- Record commands, results, supported platforms, claim limits, and residual risks in a concise
  release report.
- Verify repository documentation and artifact metadata against the release revision without moving
  private launch work into this repository.
- Create the tag and public release only after an explicit operator go-ahead and a passing
  acceptance review.

## Acceptance

- Every item in KAP-0038 release acceptance has direct evidence from the release revision.
- Release artifacts and checksums match the tagged source and pass clean-environment smoke tests.
- The owned live demo passes and leaves no kind cluster or unbounded failure artifact behind.
- Release notes state supported platforms, command surfaces, prototype format status, non-claims,
  security limits, and residual risks.
- The repository is clean, local and remote history is understood, and publication uses no force
  operation.
- The public release URL and exact revision are recorded after publication.

## Release decision

The final package and artifact version is `0.1.0`, and the intended tag is `v0.1.0`. This is the
stable, non-prerelease artifact for the experiment, not a stable-interface or production-readiness
claim. The sole supported artifact remains `x86_64-unknown-linux-gnu`. This release publishes the
GitHub source tag, archive, checksum, and notes; it does not publish a separate package registry
artifact.

## Rehearsal checklist

- [ ] Final version, MCP server version, changelog, archive name, and checksum agree.
- [ ] Repository history is understood and the exact final revision is clean.
- [ ] `cargo make fmt`, focused Clippy, `./scripts/ci-local.sh`, links, and diff hygiene pass.
- [ ] Hosted artifact smoke, reproducibility, strict clean assembly, and upload pass.
- [ ] The downloaded checksum verifies and metadata names the exact revision with
      `source_dirty: false`.
- [ ] Both explicit live-kind paths pass on the supported target and remove their cluster.
- [ ] Source-built and downloaded-artifact process-termination demonstrations pass with one apply,
      frozen receipt bytes, failed-rollout classification, and offline inspection.
- [ ] Release notes state commands, supported target, prototype formats, security limits,
      non-claims, and residual risks.
- [ ] An acceptance review passes before the explicit publication decision.
- [ ] After approval, `v0.1.0`, the public release URL, and attached archive/checksum are recorded.

## Non-goals

No production-readiness declaration, company launch narrative, adoption claim, second capability,
package registry publication, or future-expansion decision.
