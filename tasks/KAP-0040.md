# Freeze The Evaluator Application Interface

Status: active technical cursor.

Problem: the deep gateway exists, but no application-owned interface separates request-only agent
intent from operator-owned grant provisioning, trust, Kubernetes authority, signing material, and
paths.

Contract owners: [technical scope](../docs/V1.md), [architecture](../docs/ARCHITECTURE.md),
[threat model](../docs/THREAT_MODEL.md), and
[KAP-0038](../docs/experiments/KAP-0038-kubernetes-effect-gateway-boundary.md).

Depends on: [KAP-0039](KAP-0039.md).

## Work

- Name one `kapsel` application composition root over `kapsel-gateway`.
- Define the request-only caller interface for the exact Deployment image operation.
- Define bounded operator configuration for authorization trust, grant provisioning, Kubernetes
  authority, receipt signing, journal location, and receipt output.
- Decide and document how an operator provisions an exact signed grant without allowing caller input
  to appoint authority.
- Define one application result/report interface shared by future local and MCP adapters.
- Specify startup, restart, configuration, path, secret-redaction, and typed-failure behavior before
  command syntax is implemented.
- Add interface-level tests using the existing gateway rather than exposing lifecycle transitions.

## Acceptance

- Agent intent contains only stable operation identity and the four bounded operation fields.
- Operator-owned grants, trust, credentials, signing material, paths, and demo fault controls cannot
  be selected through the caller interface.
- Grant provisioning is explicit and separate from operation submission.
- The application interface owns composition but not gateway validation, lifecycle, recovery,
  classification, receipt, or publication logic.
- Configuration failures occur before persistence or Kubernetes calls and do not disclose secrets.
- Owner documents, threat cases, and tests agree before command work starts.
- Focused tests and Clippy, `./scripts/ci-local.sh`, local Markdown links, and `git diff --check`
  pass.

## Non-goals

No final CLI grammar, MCP protocol implementation, hosted process, multi-user policy system, runtime
plugin, or second capability.

## Next action

Compare a minimal file-based operator configuration with direct process arguments, then record the
smallest interface that keeps authority out of caller input.
