# Add The Thin MCP Adapter

Status: complete.

Problem: The 0.1.0 release needs one agent transport for the bounded operation, but MCP must not
become a second application interface or a generic tool runtime.

Contract owners: [MCP adapter](../docs/MCP.md), [technical scope](../docs/SCOPE.md),
[architecture](../docs/ARCHITECTURE.md), [threat model](../docs/THREAT_MODEL.md), and the
application interface produced by KAP-0040.

Depends on: [KAP-0042](KAP-0042.md).

## Work

- Read current official MCP protocol and SDK documentation before choosing dependencies or protocol
  details.
- Expose exactly one fixed-schema tool for the bounded Deployment image request.
- Convert MCP input into the same application request used by the local command.
- Convert typed application results and failures into bounded protocol responses.
- Keep operator configuration and startup outside tool input.
- Add transport conformance and application-equivalence tests.

## Acceptance

- MCP tool input contains only operation identity, namespace, Deployment, container, and immutable
  image digest.
- MCP input cannot select grants, trust, credentials, signing material, journal/output paths, fault
  controls, lifecycle transitions, shell, manifests, or arbitrary patches.
- The adapter does not reimplement validation, authorization, persistence, Kubernetes interaction,
  recovery, classification, or receipt construction.
- Local and MCP paths produce equivalent application requests and typed outcomes.
- Malformed and hostile protocol input remains bounded and secret-free.
- Focused adapter tests and Clippy, `./scripts/ci-local.sh`, local Markdown links, and
  `git diff --check` pass.

## Non-goals

No generic MCP host, tool registry, runtime plugins, remote tenancy, hosted control plane, or second
tool.

## Evidence

The fixed adapter supports MCP `2025-11-25` over newline-delimited stdio, loads the existing bounded
operator file once at startup, advertises exactly one five-field tool, converts directly into
`AgentRequest`, and maps typed application reports without changing KAP-0038 result meaning. It uses
no MCP SDK or new dependency; direct implementation with existing `serde_json` and Tokio keeps the
surface fixed and bounded.

Deterministic black-box tests cover protocol initialization and negotiation, tool listing and calls,
local/MCP equivalence, all application outcome classes, repeated execution and restart, lifecycle
ordering, request IDs, late cancellation, EOF, exact and hostile framing bounds, invalid fields,
operator authority separation, and disclosure canaries. Focused tests, all-target/all-feature
Clippy, `./scripts/ci-local.sh`, `cargo make test-demo-harness`, local Markdown links, and
`git diff --check` passed.
