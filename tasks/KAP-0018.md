# Transition Repository To Lean V1

Problem: The repository must support the profile-oriented architecture without retaining webhook product code or creating unnecessary crate/process overhead.

Acceptance:

- Create/push annotated `webhook-alpha` tag before deletion and preserve historical docs only.
- Delete webhook routes, domain models, migrations, workers, retries/circuit breakers, configuration, tests, benches, dependencies, and compatibility flags from active paths.
- Keep generic infrastructure only when already used by V1 and retaining it produces less code than replacement.
- Shape the smallest useful crates around core, verify, store, transparency, and CLI; keep profiles/adapters at the edge.
- Make `kapsel-verify` define the `ProfileValidator` interface, profile crates implement it, and the CLI act as the compile-time registration/composition root.
- Make profile crates expose producer-side `EvidenceEncoder` implementations; adapters depend on profiles, never vice versa.
- Use no runtime plugin loading in V1 and document dependency arrows unambiguously.
- Provide one containerless local/CI lane; isolate live backend and profile sandbox tests explicitly.
- Install a short owner-doc/agent spine and keep task packets tactical.
- Keep the workspace green throughout transition.

Verification:

- Compare against `webhook-alpha`; old implementation is absent, not dormant.
- Dependency graph points core <- profiles <- adapters with no domain leakage inward.
- Clean-checkout default lane passes without Docker or network services.

## Next action

Tag history, establish minimal seams, then hard-delete the webhook product.
