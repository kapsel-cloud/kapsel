# Export Portable Evidence Packages

Problem: Evidence must cross organizational boundaries with frozen one-statement semantics and enough embedded collateral for offline verification.

Acceptance:

- Export exactly one Kapsel-issued signed statement and, for witnessed packages, exactly one corresponding backend-native receipt.
- Include manifest, profile ID/version, profile collateral, disclosed/committed material, explicit omissions/deletions, untrusted verification collateral bytes, and verification instructions.
- Keep the external trust policy separate and require it as verifier input.
- Do not rely on URL references for required offline verification material.
- Treat manifest, certificates, keys, and backend metadata inside the package as untrusted evidence checked against external policy.
- Treat manifest claims as declarations checked independently by the verifier.
- Support directory and safe ZIP output.
- Refuse policy-required witnessed export without a valid receipt unless explicit unwitnessed mode is selected and labeled.
- Never include private keys.
- Delegate profile collateral and profile-specific report data through the profile boundary.

Verification:

- Reference and test profiles use the same generic exporter/verifier path.
- Zero/multiple statements, conflicting receipts, missing/deleted disclosure, altered commitments, unsafe paths, and partial packages report correctly.
- No live service or URL fetch is needed to verify an exported package.

## Next action

Build after storage and backend receipt persistence are stable.
