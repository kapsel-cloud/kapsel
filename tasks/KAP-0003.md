# Spike One SCITT Backend Path

Problem: Standards-first is useful only if one minimal external registration and offline receipt path works without becoming a general SCITT platform.

Acceptance:

- Select one signed-statement profile, receipt profile, algorithm suite, and candidate backend.
- Create and register one minimal profile-bound Kapsel statement.
- Retrieve and preserve the backend-native receipt and acknowledged submission identifier.
- Verify it offline using externally supplied trust material.
- Determine whether the backend supports one-statement/one-receipt package semantics; treat a forced multi-statement design as an explicit architecture finding, not an automatic scope expansion.
- Record backend extensions, privacy boundary, ambiguous-request recovery, idempotency, and replacement implications.
- Stop and reassess if no viable minimal backend/profile combination exists.

Verification:

- A reproducible vector verifies without live Kapsel or backend access.
- Unsupported critical parameters fail closed.
- ADR names the exact supported subset and rejects broader SCITT scope for V1.
