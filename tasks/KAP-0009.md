# Implement Minimal Statement And Key Operations

Problem: V1 needs enough producer composition and key handling to turn finalized typed evidence into exactly one accepted signed statement safely.

Acceptance:

- Define an `EvidenceEncoder` interface for typed-evidence validation, statement-payload encoding, and disclosure description.
- Compose encoder implementations at compile time in the producing application/CLI; do not add runtime plugins.
- Support issuer key generation/import, protected loading, selected COSE signing profile, and public trust-material export.
- Produce exactly one Kapsel signed statement from finalized profile evidence before storage acceptance.
- Enforce key purpose and selected algorithm/profile.
- Use explicit configured keys; ephemeral keys are test/demo-only.
- Keep private material out of bundles, logs, debug output, and backend submissions.
- Document V1 rotation, expiry, revocation, compromise, and historical-verification limitations.
- Prefer maintained libraries and require review for custom cryptographic encoding.

Verification:

- Wrong-purpose/key/algorithm and embedded-self-trust tests fail.
- Private-key leak scan passes.
- Generated statement verifies against KAP-0003 vectors and can be atomically accepted by KAP-0008.
