# Build The Evaluator Commands

Status: completed.

Problem: evaluators needed fixed prototype commands and a navigable deep product implementation
without changing public or durable behavior.

Contract owners: [build](../docs/BUILD.md), [testing](../docs/TESTING.md),
[architecture](../docs/ARCHITECTURE.md), and the application interface produced by KAP-0040.

Depends on: [KAP-0040](KAP-0040.md).

## Work

- Implement the `kapsel` executable as the one composition root.
- Implement the accepted operator grant-provisioning path.
- Implement one bounded operation command over the application interface.
- Implement a separate offline-inspection command with explicit trust, evaluation time, and limits.
- Make ordinary process restart resume eligible durable work without caller-selected lifecycle
  transitions.
- Define stable machine-readable output, bounded human output, exit status, and error-class behavior
  for this prototype release.
- Add integration tests for valid use, malformed input, untrusted grant, tuple mismatch, mutable
  image, configuration failure, restart, and offline inspection without network access.
- After command behavior is green, keep `src/main.rs` as a thin composition root and organize the
  command, application, and gateway implementations under clear module owners.
- Nest authorization, lifecycle, journal/schema/migration, concrete Kubernetes behavior,
  receipt/trust/publication, and private-seam tests under the gateway module where ownership
  requires it.
- Split long private test matrices by owned behavior and narrow internal visibility toward
  `pub(super)` without creating test-only public interfaces.

## Acceptance

- A fresh evaluator can use documented commands without calling repository-internal tests or
  sequencing durable states.
- Command input does not accept Kubernetes credentials, signing seeds, trust appointment, journal
  mutation, receipt-byte construction, fault points, shell, manifests, or arbitrary patches.
- The command and its tests cross the application interface; they do not duplicate gateway logic.
- Offline inspection performs no network access and never reports `VERIFIED`.
- Output and captured logs remain bounded and secret-free.
- `lib.rs` remains a compact exported interface map, and `main.rs` contains no gateway policy or
  lifecycle sequencing.
- The source tree makes parent operations, owned facts, and dependency direction visible without a
  crate-per-concept decomposition, generic adapter package, or shared testing package.
- The structural change preserves public types, receipt vectors, SQLite schema, durable states,
  provider-call counts, and deterministic/live proof semantics.
- Focused command/application tests and Clippy, `./scripts/ci-local.sh`, local Markdown links, and
  `git diff --check` pass.

## Non-goals

No MCP adapter, release crash harness, installer, dashboard, hosted endpoint, or
production-readiness claim.

## Evidence

- `docs/COMMANDS.md` owns the fixed prototype grammar, authority separation, output, limits, and
  exit classes.
- `tests/e2e_operation_command.rs` crosses the production executable for grant provisioning,
  malformed and mutable intent, untrusted authority, exact mismatch, unsafe paths, local HTTP
  Kubernetes mutation, ordinary restart, unrelated journal rows, bounded output, and canary
  non-disclosure.

  malformed bytes, signature and trust rejection, explicit lower limits, bounded vocabulary, and
  unavailable ambient Kubernetes/network configuration.

- `cargo test --locked -p kapsel --test e2e_inspection_command` passed.

- `cargo make check`, `cargo make test-simulation`, `cargo make fuzz-check`, `cargo make test-fuzz`,
  and `cargo make test-kind` passed after the source-locality refactor; the live gate removed its
  owned cluster.
- The final public rustdoc item set, vector hashes, and test leaf-name multiset match the committed
  command-slice baseline. The SQLite schema and provider-call assertions remain unchanged.
- A fresh read-only review found no semantic, API, SQL, receipt-byte, dependency-direction,
